## ADDED Requirements

### Requirement: جدول answer_details في قاعدة البيانات
النظام SHALL يُنشئ جدول `answer_details` في `quiz.db` عند بدء التطبيق بـ `CREATE TABLE IF NOT EXISTS`، يحتوي على الأعمدة: `id` (مفتاح أساسي)، `student_name` (اسم الطالب)، `question_id` (معرّف السؤال)، `section` (القسم)، `question_text` (نص السؤال)، `code` (الكود إن وجد)، `user_answer` (إجابة الطالب)، `correct_answer` (الإجابة الصحيحة)، `correct_text` (نص الإجابة الصحيحة)، `is_correct` (0 أو 1)، `date` (التاريخ والوقت).

#### Scenario: إنشاء الجدول تلقائيًا
- **WHEN** يبدأ التطبيق وجدول `answer_details` غير موجود
- **THEN** يتم إنشاء الجدول تلقائيًا قبل استقبال أي طلب

#### Scenario: تشغيل لاحق بجدول موجود
- **WHEN** يبدأ التطبيق وجدول `answer_details` موجود بالفعل
- **THEN** لا يتم إعادة إنشاء الجدول ولا تُفقد أي بيانات

### Requirement: حفظ تفاصيل الإجابات عند الإرسال
النظام SHALL يستدعي `save_answer_details(name, questions, answers_map, date_str)` عند إرسال الاختبار في مسار `/submit`، ليحفظ سجلًا لكل سؤال في جدول `answer_details` يشمل: إجابة الطالب، الإجابة الصحيحة، هل الإجابة صحيحة، والبيانات المرجعية للسؤال.

#### Scenario: حفظ تفاصيل طالب أكمل الاختبار
- **WHEN** يرسل الطالب الاختبار ويتم حساب النتيجة
- **THEN** يتم إدراج سجل لكل سؤال في `answer_details` يحتوي على: اسم الطالب، معرّف السؤال، القسم، نص السؤال، الكود، إجابة الطالب، الإجابة الصحيحة، نص الإجابة الصحيحة، `is_correct` (1 إذا صحيحة، 0 إذا خاطئة)، والتاريخ

#### Scenario: طالب لم يجب على سؤال
- **WHEN** يرسل الطالب الاختبار ولم يجب على سؤال معين
- **THEN** يُحفظ السجل بإجابة الطالب كـ "لم يتم الإجابة" و`is_correct = 0`

### Requirement: مسار /admin/answers لعرض قائمة الطلاب
النظام SHALL يوفر مسار `/admin/answers` (محمي بكلمة مرور الأدمن) يعرض جدولًا بكل الطلاب الذين أكملوا الاختبار من جدول `results`، مع التحقق من وجود تفاصيل في `answer_details` عبر `LEFT JOIN`. الطلاب الذين لديهم تفاصيل يظهرون مع رابط مراجعة؛ القدامى بدون تفاصيل يظهرون بعلامة `—`.

#### Scenario: عرض قائمة الطلاب
- **WHEN** يفتح الأدمن صفحة `/admin/answers` بعد تسجيل الدخول
- **THEN** يرى جدولًا بكل الطلاب المكتملين يحتوي على: الاسم، النتيجة، النسبة المئوية، التاريخ، وزر مراجعة (إن وجدت تفاصيل)

#### Scenario: طالب قديم بدون تفاصيل
- **WHEN** يوجد طالب أكمل الاختبار قبل إضافة جدول `answer_details`
- **THEN** يظهر الطالب في القائمة لكن بدون رابط مراجعة (علامة `—`)

#### Scenario: محاولة الوصول بدون صلاحية
- **WHEN** يفتح مستخدم عادي مسار `/admin/answers` بدون تسجيل دخول الأدمن
- **THEN** يُعاد توجيهه إلى صفحة تسجيل دخول الأدمن

### Requirement: مسار /admin/answers/detail لعرض تفاصيل طالب
النظام SHALL يوفر مسار `/admin/answers/detail?name=...&date=...` (محمي بكلمة مرور الأدمن) يعرض كل إجابات الطالب بالتفصيل مرتبة حسب معرّف السؤال.

#### Scenario: عرض تفاصيل طالب
- **WHEN** يضغط الأدمن على رابط المراجعة لطالب محدد
- **THEN** تظهر صفحة بكل إجابات الطالب، كل سؤال في بطاقة تعرض: رقم السؤال، القسم، حالة الإجابة (صحيحة/خاطئة)، نص السؤال، الكود إن وجد، إجابة الطالب، والإجابة الصحيحة (للإجابات الخاطئة فقط)

#### Scenario: محاولة الوصول بدون صلاحية
- **WHEN** يفتح مستخدم عادي مسار `/admin/answers/detail` بدون تسجيل دخول الأدمن
- **THEN** يُعاد توجيهه إلى صفحة تسجيل دخول الأدمن

#### Scenario: معاملات مفقودة
- **WHEN** يفتح الأدمن `/admin/answers/detail` بدون معامل `name` أو `date`
- **THEN** يُعاد توجيهه إلى `/admin/answers`
