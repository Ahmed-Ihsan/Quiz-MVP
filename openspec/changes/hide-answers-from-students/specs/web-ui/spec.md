## MODIFIED Requirements

### Requirement: صفحة النتيجة
النظام SHALL يعرض صفحة `/result` تحتوي على: اسم الطالب، النتيجة (صحيح/إجمالي)، النسبة المئوية، ورسالة تقييم بالعربية. النظام MUST **لا** يعرض للطالب أي مراجعة للإجابات الخاطئة أو الإجابات الصحيحة — هذه المعلومات تُحفظ في `answer_details` ويُمكن للأدمن فقط مراجعتها.

#### Scenario: عرض النتيجة
- **WHEN** ينتقل المستخدم إلى صفحة النتيجة بعد الإرسال
- **THEN** يرى اسمه، عدد الإجابات الصحيحة من الإجمالي، النسبة المئوية، ورسالة التقييم فقط — بدون أي تفاصيل عن الإجابات الخاطئة أو الصحيحة

#### Scenario: رسالة التقييم
- **WHEN** تظهر النتيجة
- **THEN** تظهر رسالة تقييم بالعربية حسب النسبة (مثلاً: ممتاز ≥ 90%، جيد ≥ 70%، يحتاج تحسين < 70%)

#### Scenario: عدم عرض مراجعة الإجابات
- **WHEN** توجد إجابات خاطئة
- **THEN** لا تظهر قائمة الإجابات الخاطئة ولا الإجابات الصحيحة للطالب — تُحفظ التفاصيل في `answer_details` للأدمن فقط

## ADDED Requirements

### Requirement: صفحة مراجعة إجابات الطلاب للأدمن
النظام SHALL يوفر صفحة `/admin/answers` (محمية بكلمة مرور الأدمن) تعرض قائمة بكل الطلاب الذين أكملوا الاختبار، مع رابط لمراجعة تفاصيل إجابات كل طالب. النظام SHALL يوفر أيضًا صفحة `/admin/answers/detail` تعرض كل إجابات الطالب بالتفصيل (الصحيحة والخاطئة).

#### Scenario: قائمة الطلاب في صفحة الأدمن
- **WHEN** يفتح الأدمن صفحة `/admin/answers`
- **THEN** يرى جدولًا بكل الطلاب المكتملين يحتوي على: الاسم، النتيجة، النسبة المئوية، التاريخ، وزر مراجعة (إن وجدت تفاصيل)

#### Scenario: تفاصيل إجابات طالب
- **WHEN** يضغط الأدمن على رابط مراجعة طالب
- **THEN** تظهر صفحة بكل إجابات الطالب، كل سؤال في بطاقة تعرض: رقم السؤال، القسم، حالة الإجابة (صحيحة/خاطئة)، نص السؤال، الكود إن وجد، إجابة الطالب، والإجابة الصحيحة (للإجابات الخاطئة فقط)

#### Scenario: رابط في شريط الأدمن
- **WHEN** يفتح الأدمن لوحة إدارة الأسئلة
- **THEN** يظهر رابط "إجابات الطلاب" في الشريط العلوي بجانب روابط الأدمن الأخرى
