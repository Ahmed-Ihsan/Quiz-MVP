## ADDED Requirements

### Requirement: حفظ النتائج في ملف JSON
النظام SHALL يحفظ نتيجة كل طالب في ملف `results.json` بعد إنهاء الاختبار، مع تسجيل: اسم الطالب، عدد الإجابات الصحيحة، العدد الإجمالي للأسئلة، النسبة المئوية، والتاريخ والوقت.

#### Scenario: حفظ نتيجة طالب
- **WHEN** ينهي المستخدم الاختبار ويتم حساب النتيجة
- **THEN** يتم إضافة سجل جديد إلى `results.json` يحتوي على الاسم، النتيجة، النسبة المئوية، والطابع الزمني

#### Scenario: إنشاء الملف عند عدم وجوده
- **WHEN** يتم حفظ نتيجة وملف `results.json` غير موجود
- **THEN** يتم إنشاء الملف بمصفوفة تحتوي على النتيجة الأولى

### Requirement: عرض جميع النتائج
النظام SHALL يوفر مسار `/results` يعرض جدولًا بكل نتائج الطلاب المحفوظة، مرتبة من الأحدث للأقدم.

#### Scenario: عرض جدول النتائج
- **WHEN** يفتح المستخدم صفحة `/results`
- **THEN** يظهر جدول يحتوي على: الاسم، عدد الإجابات الصحيحة، النسبة المئوية، والتاريخ

#### Scenario: لا توجد نتائج
- **WHEN** يفتح المستخدم `/results` ولا توجد نتائج محفوظة
- **THEN** تظهر رسالة بالعربية "لا توجد نتائج بعد"

### Requirement: عدم تكرار النتيجة عند التحديث
النظام MUST يمنع حفظ نتيجة مكررة لنفس الطالب في نفس الجلسة، لتجنب التكرار عند إعادة تحميل صفحة النتيجة.

#### Scenario: إعادة تحميل صفحة النتيجة
- **WHEN** يعيد المستخدم تحميل صفحة `/result`
- **THEN** لا يتم حفظ نتيجة مكررة، بل تُعرض نفس النتيجة المحفوظة في الجلسة
