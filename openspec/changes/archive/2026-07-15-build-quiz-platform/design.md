## Context

هذا مشروع جديد بدون كود موجود. المصدر الوحيد هو ملف `content.text` الذي يحتوي على بنك أسئلة بالعربية. الهدف بناء منصة اختبار تفاعلي محلية بسيطة باستخدام Flask.

## Goals / Non-Goals

### Goals
- منصة تعمل محليًا على `http://localhost:5000`
- اختبار تفاعلي بالعربية مع تصحيح تلقائي
- حفظ النتائج وعرضها
- واجهة بسيطة بدعم RTL

### Non-Goals
- لا قاعدة بيانات (نستخدم JSON فقط)
- لا نظام مصادقة معقد (اسم فقط)
- لا أسئلة عملية (مستبعدة)
- لا نشر على الإنترنت (local فقط)

## Decisions

### 1. التخزين بملفات JSON
استخدام `questions.json` للأسئلة و `results.json` للنتائج — أبسط من قاعدة بيانات ويلبي المتطلبات المحلية.

### 2. Flask بدون قاعدة بيانات
Flask مع جلسات (sessions) مبنية على cookies — كافية لتتبع اسم المستخدم بدون كلمة مرور.

### 3. عرض كل الأسئلة في صفحة واحدة
بدلًا من سؤال واحد في كل صفحة (لتجنب تعقيد JavaScript)، تُعرض جميع الأسئلة في صفحة واحدة مع التمرير. أبسط في التنفيذ والاستخدام.

### 4. استبعاد الأسئلة العملية
الأسئلة العملية (أخطاء التوصيل + تصميم مشروع) تحتاج تصحيحًا يدويًا — خارج نطاق المنصة التفاعلية.

## Risks / Trade-offs

| الخطر | الحل |
|-------|------|
| فقدان نتائج `results.json` عند إعادة التشغيل | لا يوجد حل — مقبول لمشروع محلي بسيط |
| تكرار النتيجة عند إعادة تحميل `/result` | استخدام flag في الجلسة لمنع الحفظ المكرر |
| حجم `questions.json` كبير يدويًا | استخراج آلي + مراجعة يدوية للإجابات الصحيحة |

## بنية المشروع

```
qws/
├── content.text              # الملف الأصلي (يُترك كما هو)
├── questions.json            # الأسئلة منظمة + الإجابات الصحيحة
├── results.json              # نتائج الطلاب (يُنشأ تلقائيًا)
├── app.py                    # خادم Flask
├── requirements.txt          # flask
├── openspec/                 # مواصفات OpenSpec
├── templates/
│   ├── login.html            # صفحة تسجيل الدخول
│   ├── quiz.html             # صفحة الاختبار
│   ├── result.html           # صفحة النتيجة + المراجعة
│   └── results.html          # جدول نتائج الطلاب
└── static/
    ├── style.css             # تنسيق RTL + تصميم متجاوب
    └── script.js             # التحقق من الإجابات قبل الإرسال
```

## تدفق البيانات

```
المستخدم → / (إدخال الاسم) → جلسة Flask
         → /quiz (عرض الأسئلة من questions.json)
         → /submit (تصحيح + حساب النتيجة)
         → حفظ في results.json
         → /result (عرض النتيجة + مراجعة الأخطاء)
         → /results (عرض كل النتائج)
```
