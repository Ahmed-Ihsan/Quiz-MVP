# web-ui Delta — تحسين متقدم للواجهة

## ADDED Requirements

### Requirement: نظام تصميم بـ CSS Custom Properties
النظام MUST يستخدم CSS Custom Properties (متغيرات CSS) لجميع الألوان والمسافات والظلال
ونصف القطر ومدة الانتقالات، موزّعة على وضعين: نهاري (`:root`) وليلي (`[data-theme="dark"]`).

#### Scenario: تبديل الوضع الليلي
- **WHEN** يضغط المستخدم زر تبديل الوضع الليلي
- **THEN** يتبدّل لون كامل الواجهة فوراً بين النهاري والليلي، ويُحفظ التفضيل في `localStorage`

#### Scenario: القيمة الافتراضية للوضع
- **WHEN** يفتح المستخدم المنصة لأول مرة (لا توجد قيمة محفوظة)
- **THEN** يُحدّد الوضع تلقائياً من `prefers-color-scheme` دون وميض (FOUC)

### Requirement: خط عربي ويب احترافي
النظام MUST يستخدم خط Cairo من Google Fonts لجميع النصوص العربية، مع `line-height: 1.75`
للنصوص و`letter-spacing: 0` إلزامي، و`font-display: swap` لمنع حجب النص أثناء التحميل.

#### Scenario: عرض الخط العربي
- **WHEN** تُحمّل أي صفحة
- **THEN** يظهر النص العربي بخط Cairo، بارتفاع سطر 1.75، وتباعد أحرف صفري

### Requirement: الوصولية (WCAG)
النظام MUST يوفر: `aria-label` عربية على الأزرار والمؤشرات، مؤشرات `focus-visible` واضحة
(حلقة 3px)، `aria-live="polite"` لشريط التقدم، واحترام `prefers-reduced-motion` لتعطيل
الحركات. تباين الألوان MUST يحقق WCAG AA (4.5:1 للنص العادي).

#### Scenario: التنقل بلوحة المفاتيح
- **WHEN** يتنقل المستخدم بالـ Tab بين العناصر
- **THEN** يظهر مؤشر تركيز واضح (حلقة 3px بلون primary) حول العنصر النشط

#### Scenario: تعطيل الحركات
- **WHEN** يكون `prefers-reduced-motion: reduce` مفعّل في النظام
- **THEN** تُعطّل جميع الانتقالات والأنميشن (مدة 0.01ms)

### Requirement: إشعارات Toast بدل alert
النظام SHALL يستبدل `alert()` و`confirm()` الأصليين بإشعارات toast منزلقة ومودالات
مخصصة أنيقة بالعربية، مع `role="alert"` و`aria-live` للوصولية.

#### Scenario: تنبيه تبديل التبويب
- **WHEN** يكتشف النظام مغادرة الصفحة أثناء الاختبار
- **THEN** يظهر toast تحذيري منزلق من الأعلى يختفي تلقائياً بعد 4 ثوانٍ (بدل alert)

#### Scenario: تأكيد الأسئلة غير المُجابة
- **WHEN** يضغط المستخدم "إنهاء الاختبار" وتبقى أسئلة بدون إجابة
- **THEN** يظهر مودال مخصص يعرض عدد الأسئلة غير المُجابة مع زرَّي "متابعة" و"إلغاء"

### Requirement: تفاعلات دقيقة في صفحة الاختبار
النظام SHALL يوفر تفاعلات دقيقة (micro-interactions) بـ CSS خالص: راديو مخصص بأنميشن
اختيار، تأكيد بصري فوري للخيارات، ظهور تدريجي لبطاقات الأسئلة عند التمرير، وحالة loading
لزر الإرسال.

#### Scenario: اختيار خيار
- **WHEN** يختار المستخدم خياراً في سؤال
- **THEN** تظهر نقطة في دائرة الراديو بأنميشن، تتلوّن حدود الخيار، وتظهر خلفية خفيفة

#### Scenario: ظهور بطاقات الأسئلة
- **WHEN** تمر بطاقة سؤال في نطاق العرض (viewport)
- **THEN** تظهر بتأثير `fade-in-up` سلس (300ms ease-out)

#### Scenario: إرسال الاختبار
- **WHEN** يضغط المستخدم "إنهاء الاختبار" ويُؤكد
- **THEN** يدخل الزر حالة loading (spinner + "جاري الإرسال...") ويُعطّل لمنع الإرسال المزدوج

## MODIFIED Requirements

### Requirement: صفحة الاختبار
النظام SHALL يعرض صفحة `/quiz` تحتوي على: رأس زجاجي (glassmorphism) ثابت مع شريط تقدم
بتدرج لوني ونسبة مئوية، مؤقت بتصميم pill محسّن، زر تبديل الوضع الليلي، زر ملء الشاشة
بأيقونة، اسم الطالب مع avatar دائري، جميع الأسئلة في بطاقات بظلال طبقية وأرقام بشارات
دائرية، خيارات براديو مخصص وتفاعلات دقيقة، صندوق رمز استئناف مع زر نسخ، وزر "إنهاء
الاختبار" بحالة loading.

#### Scenario: شريط التقدم
- **WHEN** يتصفح المستخدم صفحة الاختبار
- **THEN** يظهر شريط تقدم بتدرج لوني (primary → accent) يعرض النسبة المئوية رقمياً مع animation سلس

#### Scenario: زر إنهاء الاختبار
- **WHEN** يصل المستخدم إلى زر الإرسال
- **THEN** يرى زراً أحمر واضحاً يعرض حالة loading عند الضغط مع تعطيل لمنع الإرسال المزدوج

### Requirement: التنسيق والتصميم البسيط
النظام SHALL يستخدم ملف `static/style.css` بنظام تصميم حديث: CSS Custom Properties، خط
Cairo عربي، ظلال طبقية ناعمة، تصميم متجاوب، ودعم الوضع الليلي.

#### Scenario: التصميم المتجاوب
- **WHEN** يفتح المستخدم المنصة من جوال
- **THEN** تتكيف الصفحات تلقائياً مع عرض الشاشة مع الحفاظ على الوصولية والتباين

### Requirement: JavaScript للتحقق من الإجابات
النظام SHALL يستخدم ملف `static/script.js` للتحقق من الأسئلة غير المُجابة قبل الإرسال
عبر مودال مخصص (بدل confirm)، إدارة الوضع الليلي، إشعارات toast، التنقل بلوحة المفاتيح،
والظهور التدريجي للبطاقات.

#### Scenario: التحقق قبل الإرسال
- **WHEN** يضغط المستخدم "إنهاء الاختبار" وتبقى أسئلة بدون إجابة
- **THEN** يظهر مودال مخصص (بدل confirm) يعرض عدد الأسئلة غير المُجابة مع زرَّي تأكيد وإلغاء
