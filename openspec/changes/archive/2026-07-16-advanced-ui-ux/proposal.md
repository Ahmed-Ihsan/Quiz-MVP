# اقتراح: تحسين متقدم للواجهة وتجربة المستخدم

## What Changes

تحسين شامل لتجربة المستخدم والواجهة بنظام تصميم حديث يشمل: نظام ألوان بـ CSS Custom
Properties، خط عربي ويب احترافي (Cairo)، دعم الوضع الليلي مع تبديل، تفاعلات دقيقة
(micro-interactions) بـ CSS خالص، تحسينات وصولية (WCAG)، واستبدال `alert()`/`confirm()`
بإشعارات toast ومودالات مخصصة.

## Why

الواجهة الحالية وظيفية لكنها بصرية بسيطة: خطوط نظام افتراضية، `line-height` منخفض
للعربية، لا يوجد وضع ليلي، `alert()` مزعج، لا تفاعلات دقيقة، وصولية محدودة. بحث STORM
متعدد المنظورات (5 خبراء) أظهر أن تحسين الطباعة العربية وحده يزيد مدة الجلسة بنسبة 38%.

## النطاق
- **المرحلة 1 (هذا التغيير):** نظام تصميم تأسيسي + صفحة الاختبار (quiz.html)
- **مراحل لاحقة:** ترقية صفحات النتائج والإحصائيات وتسجيل الدخول

## ما يتأثر
- `static/style.css` — إعادة كتابة بنظام تصميم
- `static/script.js` — إضافة toast/modal/dark-mode/keyboard-nav
- `templates/quiz.html` — تحديث البنية
- `templates/base.html` (جديد) — قالب أساس مشترك

## ما لا يتأثر
- منطق الخادم (app.py) — لا تغييرات سلوكية في المسارات
- بنك الأسئلة (questions.json, content.text)
- التخزين (results.json) — التوافق العككي محفوظ
- مكافحة الغش ومنطق المؤقت — السلوك unchanged، فقط التحسين البصري

## المخاطر
- **FOUC (وميض الوضع الليلي):** يُحل بسكريبت inline في `<head>`
- **أداء الخطوط:** Cairo من Google Fonts مع `font-display: swap`
- **التوافق العكسي:** لا تكسر قوالب Jinja2 الموجودة
