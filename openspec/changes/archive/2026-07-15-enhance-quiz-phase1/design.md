## Context

منصة الاختبار الحالية (المرحلة 0 — `build-quiz-platform`) منفّذة ومؤرشفة. توفر: تسجيل دخول بالاسم، عرض 110 سؤال اختيارات في صفحة واحدة `/quiz`، تصحيح تلقائي، حفظ النتائج في `results.json`، وعرضها في `/results`. التقنيات: Flask + HTML/CSS/JS عادي + ملفات JSON (بدون قاعدة بيانات، بدون framework أمامي).

هذه المرحلة الأولى (`enhance-quiz-phase1`) تضيف ميزات عالية القيمة بحثتُها عبر منهجية STORM في منصات الاختبار الشائعة (Kahoot، Quizizz، Google Forms، FormNX، QuizFactory، EasyEvaluate، FlexiQuiz)، مع الالتزام الصارم بسهولة الاستخدام وعدم تغيير بنية البيانات الجوهرية أو تجربة التنقل.

## Goals / Non-Goals

**Goals:**
- مؤقت زمني تنازلي للاختبار كامل مع إرسال تلقائي عند انتهاء الوقت (قابل للتعطيل).
- حماية أساسية من الغش: تعطيل نسخ/لصق، كشف تبديل التبويب، تعطيل قائمة السياق، وضع ملء الشاشة الاختياري.
- تصدير نتائج الطلاب إلى CSV (UTF-8 BOM) يفتحه Excel مباشرة.
- لوحة إحصائيات بسيطة `/stats`: ملخص أداء + أصعب الأسئلة + توزيع الدرجات.
- توافق عكسي مع سجلات `results.json` القديمة (الحقول الجديدة اختيارية).

**Non-Goals:**
- لا تغيير لبنية `questions.json` (يؤجل للمرحلة 4 — أنواع أسئلة متعددة ووسائط).
- لا تغيير لتجربة التنقل "عرض كل الأسئلة في صفحة واحدة" (يؤجل للمرحلة 3 — تنقل سؤال-بسؤال).
- لا خلط الأسئلة/الخيارات (يؤجل للمرحلة 2 — يتطلب تحديث مواصفة "الترتيب الثابت").
- لا مصادقة بكلمة مرور (يؤجل للمرحلة 2).
- لا حفظ تلقائي للتقدم عبر الجلسات (يؤجل للمرحلة 5).
- لا شهادة PDF (يؤجل للمرحلة 3).
- لا تبعيات خارجية جديدة — تصدير CSV بوحدة `csv` المدمجة في Python.

## Decisions

### 1. المؤقت في الواجهة الأمامية + طابع زمني في الخادم
المؤقت التنازلي يُدار بـ JavaScript في المتصفح (لتحديث العرض كل ثانية). وقت بدء الاختبار يُحفظ في جلسة Flask عند فتح `/quiz` لحساب `time_spent_seconds` بدقة في الخادم عند الإرسال — لا الاعتماد على وقت العميل (قابل للتلاعب).

**السبب:** المؤقت في الواجهة للعرض الفوري، والطابع الزمني في الخادم للدقة ومنع التلاعب.

### 2. الإرسال التلقائي عبر نفس نموذج `/submit`
عند انتهاء الوقت، يستدعي JavaScript `form.submit()` لنفس نموذج الاختبار — لا مسار جديد. هذا يعيد استخدام منطق التصحيح الموجود بالكامل.

### 3. قابلية التعطيل عبر متغير إعداد
`QUIZ_TIMER_MINUTES = 30` في أعلى `app.py`. عند ضبطه على 0، لا يُمرّر المؤقت إلى القالب ولا يُعرض. أبسط من ملف إعداد منفصل ويلبي طبيعة المشروع المحلية.

### 4. كشف تبديل التبويب بأحداث المتصفح
استخدام `document.visibilitychange` و`window.blur` لكشف مغادرة الصفحة. يُحفظ العدد في حقل مخفي (hidden input) يُرسل مع النموذج. لا يُمنع التبديل تمامًا (قد يغلق المستخدم عن طريق الخطأ) — فقط تنبيه + تسجيل.

### 5. تصدير CSV بترميز UTF-8 with BOM
وحدة `csv` المدمجة + كتابة BOM (`\ufeff`) في بداية الملف لضمان ظهور العربية في Excel. يُرجع Flask الملف عبر `Response` مع `mimetype='text/csv'` و`Content-Disposition: attachment`.

### 6. لوحة الإحصائيات تحسب من `results.json` مباشرة
لا تخزين مسبق للإحصائيات — تُحسب عند كل طلب لـ `/stats` من `results.json`. الأداء كافٍ لمشروع محلي بعدد طلاب محدود. أصعب الأسئلة تُحسب من حقل `wrong_question_ids` المُضاف لكل سجل.

### 7. توافق عكسي مع السجلات القديمة
عند تحميل السجلات، تُعامَل الحقول المفقودة (`time_spent_seconds`, `tab_switches`, `wrong_question_ids`) كقيم افتراضية (0 / مصفوفة فارغة) عبر `r.get("field", default)`. لا ترحيل للبيانات القديمة.

## Risks / Trade-offs

| الخطر | الحل |
|-------|------|
| تلاعب المستخدم بالمؤقت عبر أدوات المطور | المؤقت للعرض فقط؛ وقت البدء يُحفظ في الخادم لحساب الوقت الفعلي |
| كشف تبديل التبويب قد يعطي إنذارات كاذبة (إشعارات النظام) | تنبيه فقط لا منع؛ العدد يُسجّل للمراجعة لا للعقاب التلقائي |
| نمو `results.json` مع `wrong_question_ids` | مقبول لمشروع محلي بعدد طلاب محدود؛ كل سجل يضيف ~110 رقم |
| إحصائيات تُحسب في كل طلب | كافٍ للأعداد المحلية؛ يمكن إضافة تخزين مؤقت لاحقًا إن لزم |
| Excel قد يفسد العربية بدون BOM | كتابة BOM صراحةً في بداية ملف CSV |

## بنية الملفات المتأثرة

```
qws/
├── app.py                    # + QUIZ_TIMER_MINUTES, /stats, /export, تمديد /submit
├── results.json              # + حقول اختيارية جديدة (توافق عكسي)
├── templates/
│   ├── quiz.html             # + المؤقت، زر ملء الشاشة
│   ├── results.html          # + زر تصدير CSV، رابط /stats
│   └── stats.html            # جديد — لوحة الإحصائيات
└── static/
    ├── script.js             # + المؤقت، كشف التبويب، تعطيل نسخ/لصق، ملء الشاشة
    └── style.css             # + تنسيق المؤقت، اللوحة، التوزيع
```

## تدفق البيانات (الإضافات)

```
المستخدم → /quiz (حفظ quiz_start_time في الجلسة + المؤقت في الواجهة)
         → تبديل تبويب؟ → تنبيه + زيادة عداد (hidden input)
         → انتهاء الوقت؟ → form.submit() تلقائيًا
         → /submit (حساب time_spent_seconds من الجلسة + tab_switches من النموذج)
         → حفظ في results.json (مع wrong_question_ids, time_spent_seconds, tab_switches)
         → /result

المعلم → /results (جدول + زر تصدير + رابط /stats)
       → /export (تنزيل CSV بترميز UTF-8 BOM)
       → /stats (ملخص + أصعب الأسئلة + توزيع الدرجات)
```
