## MODIFIED Requirements

### Requirement: حفظ النتائج في ملف JSON
النظام SHALL يحفظ نتيجة كل طالب في قاعدة بيانات SQLite محلية (`quiz.db`) بعد إنهاء الاختبار، مع تسجيل: اسم الطالب، عدد الإجابات الصحيحة، العدد الإجمالي للأسئلة، النسبة المئوية، التاريخ والوقت، الوقت المستغرق بالثواني (`time_spent_seconds`)، عدد مرات تبديل التبويب (`tab_switches`)، وقائمة معرّفات الأسئلة الخاطئة (`wrong_question_ids`) لدعم إحصائيات أصعب الأسئلة. الكتابة MUST تكون ذرية (atomic commit) لمنع التلف عند المقاطعة.

#### Scenario: حفظ نتيجة طالب
- **WHEN** ينهي المستخدم الاختبار ويتم حساب النتيجة
- **THEN** يتم إضافة سجل جديد إلى جدول النتائج في `quiz.db` يحتوي على: الاسم، النتيجة، النسبة المئوية، الطابع الزمني، `time_spent_seconds`، `tab_switches`، و`wrong_question_ids` (مصفوفة معرّفات الأسئلة الخاطئة مخزّنة كـ JSON)

#### Scenario: إنشاء قاعدة البيانات عند عدم وجودها
- **WHEN** يتم حفظ نتيجة وملف `quiz.db` (أو الجدول) غير موجود
- **THEN** يتم إنشاء قاعدة البيانات والجدول تلقائيًا ثم حفظ النتيجة الأولى

#### Scenario: توافق عكسي مع السجلات القديمة
- **WHEN** يُحمّل النظام السجلات من SQLite ويحتوي على سجلات قديمة بدون الحقول الجديدة
- **THEN** يعمل النظام بدون خطأ، وتُعامَل الحقول المفقودة كقيم افتراضية (0 للحقول الرقمية، مصفوفة فارغة لـ `wrong_question_ids`)

#### Scenario: ترحيل results.json الحالي عند أول تشغيل
- **WHEN** يتم تشغيل التطبيق وملف `results.json` موجود وجدول النتائج في SQLite فارغ
- **THEN** يتم استيراد جميع سجلات `results.json` إلى SQLite مرة واحدة، وتُحافَظ على جميع البيانات دون فقدان

### Requirement: تصدير النتائج إلى CSV
النظام SHALL يوفر مسار `/export` يصدّر جميع نتائج الطلاب المحفوظة في قاعدة SQLite كملف CSV قابل للفتح في Excel وGoogle Sheets، مع رؤوس أعمدة بالعربية.

#### Scenario: تصدير النتائج
- **WHEN** يفتح المستخدم مسار `/export`
- **THEN** يُنزّل المتصفح ملف CSV يحتوي على أعمدة: الاسم، عدد الإجابات الصحيحة، العدد الإجمالي، النسبة المئوية، التاريخ، الوقت المستغرق، عدد تبديلات التبويب

#### Scenario: ترميز CSV للعربية
- **WHEN** يُفتح ملف CSV المُصدّر في Excel
- **THEN** تظهر الأحرف العربية بشكل صحيح (ترميز UTF-8 مع BOM) دون رموز مشوّهة

#### Scenario: لا توجد نتائج للتصدير
- **WHEN** يفتح المستخدم `/export` ولا توجد نتائج محفوظة
- **THEN** يُنزّل ملف CSV يحتوي على صف الرؤوس فقط

## ADDED Requirements

### Requirement: تهيئة قاعدة البيانات تلقائيًا
النظام SHALL يهيّئ قاعدة بيانات SQLite والجدول المطلوب تلقائيًا عند بدء التطبيق، دون الحاجة إلى خطوة إعداد يدوية.

#### Scenario: أول تشغيل بدون قاعدة بيانات
- **WHEN** يبدأ التطبيق وملف `quiz.db` غير موجود
- **THEN** يتم إنشاء ملف `quiz.db` وجدول النتائج بالأعمدة المطلوبة تلقائيًا قبل استقبال أي طلب

#### Scenario: تشغيل لاحق بقاعدة موجودة
- **WHEN** يبدأ التطبيق وملف `quiz.db` موجود بالفعل
- **THEN** لا يتم إعادة إنشاء الجدول ولا تُفقد أي بيانات موجودة
